How to Use This Model

Follow these steps to integrate and start getting value from this model in minutes.

1
Get API Access

Sign up for a NexusAI account (free). Navigate to Settings → API Keys and create a new key. Your key grants access to all models in the marketplace — no separate accounts needed.

2
Choose your integration method

Options: (a) NexusAI REST API — simple HTTP requests from any language, (b) Official SDK — Python, Node.js, Go packages available, (c) No-code — use the built-in Playground or connect via Zapier / Make.

Quick Start (Python)
import nexusai client = nexusai.Client(api_key="YOUR_KEY") response = client.chat( model="gpt-4o", messages=[{"role":"user","content":"Hello!"}] ) print(response.content)
3
Understand input and output formats

This model accepts text, images, and PDFs as input. Outputs are text and structured JSON. The context window is 128K tokens — roughly 96,000 words. For long documents, consider chunking content into sections.

4
Set parameters for your use case

Key parameters: temperature (0 = deterministic, 1 = creative), max_tokens (controls output length), system (sets model persona and behaviour). Start with temperature 0.3–0.7 for most applications.

5
Test in the Playground first

Before writing code, iterate on your prompt in the built-in Playground. Test edge cases, adjust tone and format, then copy the final prompt into your integration.

Pro tip: Start with a small free-tier experiment. Build a minimal working version, measure quality and latency, then scale. Most production apps iterate through 3–5 prompt versions before going live.

Prompt Engineering for GPT-4o

Well-crafted prompts dramatically improve model output quality. Follow these principles to get the best results every time.

Principle 1 — Be explicit about format
Summarize the following text in exactly 3 bullet points. Each bullet should be one sentence, under 20 words. Text: {your_text_here}
Principle 2 — Assign a role
You are a senior software engineer specializing in Python. Review the following code for bugs, performance issues, and style violations. Be concise and actionable. Code: {your_code_here}
Principle 3 — Chain-of-thought for complex tasks
Solve this step by step, showing your reasoning at each stage. Problem: {your_problem_here} Think through: assumptions → approach → calculation → answer
Principle 4 — Few-shot examples
Classify customer sentiment. Examples: Input: "Shipping was fast!" → Output: positive Input: "Product broke after a day." → Output: negative Input: "It's okay, nothing special." → Output: neutral Now classify: "{new_review_here}"

Pro tips: Always specify the desired output length. Use delimiters like triple backticks to separate instructions from content. For JSON output, include a sample structure in the prompt.

Create an Agent with GPT-4o

Follow these steps to build a powerful AI agent in under 10 minutes.

1
Define your agent's purpose

Clearly state what your agent should do. Example: "A customer support agent that answers product questions, escalates billing issues, and creates Jira tickets for bugs."

2
Write the system prompt

The system prompt defines the agent's persona, scope, and behaviour. Be explicit about what the agent should and shouldn't do. Include tone, response length, and escalation rules.

3
Connect tools & APIs

Equip your agent with tools: web search, database lookup, email sender, calendar API, Slack webhook. GPT-4o supports function calling — define your tools in JSON schema format.

4
Set up memory

Configure short-term (conversation history) and long-term memory (vector store). This lets the agent remember user preferences and important context across sessions.

5
Test & iterate

Run the agent through 20+ test scenarios covering edge cases. Refine the system prompt based on failures. Use our Agent Playground to debug and tune before deployment.

6
Deploy & monitor

Get a shareable endpoint or embed widget. Monitor performance in the NexusAI dashboard — track response quality, latency, token usage, and user satisfaction scores in real time.
