# NexusAI Hub Backend Guide

This file provides working guidance for Claude when editing the backend in this repository.

## Project Overview

- Framework: NestJS (`src/`)
- Language: TypeScript
- Database: MongoDB via Mongoose
- Sessions: `express-session` + `connect-mongo`
- API docs: Swagger at `/api/docs`

## Run Commands

- Install: `npm install`
- Dev server: `npm run start:dev`
- Build: `npm run build`
- Lint: `npm run lint`
- Test: `npm run test`

## Environment Variables

- `PORT` (default: `4000`)
- `MONGO_URI` (primary MongoDB connection string)
- `SESSION_SECRET` (required for secure session cookies)
- `CORS_ORIGIN` (frontend origin, e.g. `http://localhost:3000`)

Backward compatibility:
- `MONGODB_URI` is also accepted as a fallback in code.

## Backend Architecture

- `src/main.ts`
  - app bootstrap, CORS, global validation pipe, Swagger setup
  - session middleware + Mongo session store setup
- `src/app.module.ts`
  - root module, MongoDB connection, feature module registration
- `src/catalog/`
  - endpoints for frontend static data:
    - `/api/catalog/labs`
    - `/api/catalog/models`
    - `/api/catalog/agents`
- `src/chat-hub/`
  - endpoint for chat-hub configuration:
    - `/api/chat-hub`
- `src/auth/`
  - auth + session APIs:
    - `/api/auth/signup`
    - `/api/auth/login`
    - `/api/auth/guest`
    - `/api/auth/session`
    - `/api/auth/logout`
  - user schema in `src/auth/schemas/user.schema.ts`
- `src/data/static-data.ts`
  - JSON payloads mirrored from frontend data files

## API Conventions

- Global API prefix is `/api`.
- Return JSON responses only.
- Keep response shapes stable to avoid breaking frontend.
- For new request DTOs, use `class-validator` decorators.
- For new endpoints, add Swagger decorators (`@ApiTags`, `@ApiOperation`, etc.).

## Auth and Session Rules

- Session user is stored in `session.user`.
- Guest mode must set `guestMode: true`.
- Logged user mode must set `guestMode: false`.
- Sessions must persist in MongoDB (do not switch to in-memory store).
- Cookies should remain `httpOnly`; keep secure flag tied to production mode.

## Data Consistency Rules

- Treat frontend data shape as contract.
- If updating `src/data/static-data.ts`, preserve key names used by frontend.
- Avoid renaming existing response fields unless frontend is updated in the same task.

## Security Notes

- Never hardcode secrets or production credentials in source files.
- Use env vars for DB URI and session secret.
- Password handling currently uses SHA-256 hashing for baseline support; if auth hardening is requested, migrate to bcrypt/argon2 and update login/signup flow accordingly.

## Validation Checklist (before finishing)

1. `npm run build` passes.
2. Modified endpoints appear in Swagger (`/api/docs`).
3. MongoDB connection works with `MONGO_URI`.
4. Guest and logged session flows still work (`/api/auth/guest`, `/api/auth/login`, `/api/auth/session`).

## Notes

- This backend currently serves static catalog/chat data and simple session auth for frontend integration.
- Prefer incremental changes and avoid unrelated refactors.
