# NexusAI Hub Frontend Guide

This file provides working guidance for Claude when editing the frontend in this repository.

## Project Overview

- Framework: Next.js App Router (`src/app`)
- Language: TypeScript + React
- State: Redux Toolkit (`src/store`)
- Styling: Tailwind utility classes + project design tokens in `globals.css`
- Motion/UI: `framer-motion`, `react-icons`

## Run Commands

- Install: `npm install`
- Dev server: `npm run dev`
- Lint: `npm run lint`
- Build: `npm run build`

## Frontend Architecture

- `src/app/`
  - `page.tsx` = landing page
  - `chat/page.tsx`, `marketplace/page.tsx`, `agents/page.tsx`, `research/page.tsx` = routed app views
  - `login/page.tsx`, `signup/page.tsx` = auth pages
- `src/components/`
  - `landing/*` = public landing UI
  - `app/*` = authenticated/app workspace UI
  - `auth/*` = sign in/up UI and shell
  - `shared/*` = reusable modal/toast UI
- `src/store/`
  - `appSlice.ts` = global app view state + toasts
  - `chatSlice.ts` = chat messages, onboarding state, file attachments
  - `authSlice.ts` = auth session/loading/error state
  - `modalSlice.ts` = model modal state

## Routing Conventions

- Use URL routes for navigation (not only Redux state changes).
- Main pages:
  - `/`
  - `/chat`
  - `/marketplace`
  - `/agents`
  - `/research`
  - `/login`
  - `/signup`
- Prefer `router.push(...)` for user navigation actions.

## Auth Flow Rules

- Auth session is persisted in `localStorage` under `nexusai:user`.
- Hydration of persisted auth happens in `src/providers/StoreProvider.tsx`.
- `login` and `signup` pages support `next` query param and redirect authenticated users away.
- Keep `next` URL validation safe (must be app-relative).

## Chat Hub Rules

- `ChatInput.tsx` supports:
  - text prompt
  - mic voice dictation (browser speech recognition)
  - file/image attachment selection
- Message schema supports `attachments` metadata (name/type/size).
- When extending file handling, preserve current UX:
  - show attached file chips before send
  - allow removing individual files
  - show attachment labels in user bubbles after send

## Code Style Expectations

- Keep changes minimal and scoped.
- Reuse existing patterns/components before creating new abstractions.
- Maintain current UI tone and spacing scale.
- Do not introduce unrelated refactors.
- Use strict TypeScript types (avoid `any` unless unavoidable).

## Validation Checklist (before finishing)

1. `npm run lint` passes.
2. Changed route paths load in dev mode.
3. No regression in:
   - navbar route navigation
   - login/signup redirect behavior
   - chat send flow and attachment UX

## Notes

- This frontend currently uses mock/local chat responses and local auth simulation.
- If backend APIs are added, keep adapters isolated in dedicated service modules and avoid coupling UI components directly to fetch logic.
